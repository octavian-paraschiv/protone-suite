This page of the Settings Dialog is used to configure (activate/deactivate) the integration with the Windows Explorer.

When the integration is activated, the user can also be used to define which file types are handled by the application:

  • Audio files: The user can select/deselect individual audio files, or all audio files can be selected/deselected by using the buttons at the bottom of this panel
  • Video files: The user can select/deselect individual video files, or all video files can be selected/deselected by using the buttons at the bottom of this panel
  • Playlist files: The user can select/deselect individual playlist files, or all playlist files can be selected/deselected by using the buttons at the bottom of this panel

When the Windows Explorer Integration is activated, the Windows Explorer (or any other application that implements the Windows Shell interfaces, e.g. Total Commander or Free Commander) will add the options "Play with ProTONE player" or "Enqueue in ProTONE player" to the context menu shown when a file system entry is right-clicked in Windows Explorer, like in the screenshot below:

These context menu entries are hence shown:

  • When the Windows Explorer Integration is activated, AND ...
  • The selected entries represent valid media entries, meaning that:
    • The entry is a valid media file (i.e. it is checked in the corresponding list in this panel)
    • The entry is a disk or a folder that contain valid media files, or one of its subfolders (at most 3 levels deep) contains valid media files
    • The entry is an Audio CD disk, or a Video DVD disk
Also, the user can define how should the application open the media files that are launched (that is - double clicked with mouse, or selected then pressed Enter key) from Windows Explorer. The options are to:
  • Enqueue the files at the end of current playlist (default option)
  • Play the files, replacing current playlist
