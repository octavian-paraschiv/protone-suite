This dialog is displayed whenever the user presses the "Show log file" shortcut key (F12). Typically, this window is brought up in the process of troubleshooting the application. This window contains a list of entries that is updated in real time.

The toolbar has the following fields:
* The first toolbar field is disabled and it displays the name of the current log file.
* The second toolbar field defines how many log entries are displayed in the list.
* The next four toolbar fields defines which log entry types are displayed in the list: Trace, Info, Warning, Error.
* The 7th toolbar field is used to save the contents of the log list or the entire log file to a custom file on the disk.
* The 8th toolbar field is used to freeze the contents of the log list (that is, stop updating the list in real time).
* The 9th toolbar field is used to purge the contents of the log file.

The log entry list has the following columns:
* The first column identifies the entry type: Trace, Info, Warning, Error.
* The second column represents the timestamp (when did the respective event occur).
* The 3rd and the 4th columns are the Process ID and the Thread ID.
* The 5th column identifies in which application module did the event occur.
* The 6th column describes the message associated with the event.
Double-clicking an entry in the list will display additional event information (if available).
